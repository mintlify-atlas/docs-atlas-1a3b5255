## Overview

This guide walks you through linking attribution effects using the Carino method. You'll learn how to prepare your data, run the linking, and interpret the results.

<Steps>
  <Step title="Import the package">
    Import attriblink and pandas for data handling:
    
    ```python
    import pandas as pd
    from attriblink import link
    ```
  </Step>
  
  <Step title="Prepare your data">
    Create time series for portfolio returns, benchmark returns, and attribution effects. All data must have aligned indices.
    
    ```python
    # Quarterly portfolio and benchmark returns
    portfolio_returns = pd.Series(
        [0.025, 0.035, -0.012, 0.048],
        index=pd.date_range("2025-01-01", periods=4, freq="ME")
    )
    benchmark_returns = pd.Series(
        [0.018, 0.028, -0.015, 0.038],
        index=portfolio_returns.index
    )
    
    # Attribution effects from Brinson-Fachler analysis
    effects = pd.DataFrame({
        "allocation":    [0.005, 0.006, 0.002, 0.008],
        "selection":     [0.003, 0.002, -0.001, 0.004],
        "interaction":   [0.001, 0.001, 0.000, 0.002]
    }, index=portfolio_returns.index)
    ```
    
    <Tip>
    Effects must sum to excess return (portfolio - benchmark) for each period. Attriblink validates this by default and warns you if there's a mismatch.
    </Tip>
  </Step>
  
  <Step title="Link the effects">
    Apply the Carino linking method:
    
    ```python
    # Link effects using Carino method
    result = link(effects, portfolio_returns, benchmark_returns)
    ```
    
    The `link()` function validates your inputs and applies the Carino method to compute linked effects.
  </Step>
  
  <Step title="View the results">
    Access linked effects and print a summary:
    
    ```python
    # View formatted summary
    print(result.summary())
    
    # Access individual linked effects
    print(f"Allocation: {result['allocation']:.4%}")
    print(f"Selection:  {result['selection']:.4%}")
    
    # Check the k-factor
    print(f"k-factor: {result.k_factor:.4f}")
    ```
  </Step>
</Steps>

## Understanding the Output

The `AttributionResult` object provides several ways to access your results:

### Summary View

The `summary()` method displays a formatted table showing:
- Period-by-period effects
- Total effects (sum across periods)
- Linked effects (geometrically adjusted)
- Cumulative excess return

### Individual Effects

Access linked effects by name using bracket notation:

```python
allocation_effect = result['allocation']
selection_effect = result['selection']
```

### k-Factor Interpretation

The k-factor indicates return volatility patterns:

- **k = 1.0**: Arithmetic and geometric returns match (no adjustment needed)
- **k > 1.0**: Volatile excess returns — effects are scaled up
- **k < 1.0**: Consistent excess returns — effects are scaled down

<Note>
The sum of linked effects **always equals** the cumulative excess return, ensuring geometric additivity.
</Note>

## Complete Example

Here's the full working example:

<CodeGroup>
```python Basic Usage
import pandas as pd
from attriblink import link

# Quarterly portfolio and benchmark returns
portfolio_returns = pd.Series(
    [0.025, 0.035, -0.012, 0.048],
    index=pd.date_range("2025-01-01", periods=4, freq="ME")
)
benchmark_returns = pd.Series(
    [0.018, 0.028, -0.015, 0.038],
    index=portfolio_returns.index
)

# Attribution effects from Brinson-Fachler
effects = pd.DataFrame({
    "allocation":    [0.005, 0.006, 0.002, 0.008],
    "selection":     [0.003, 0.002, -0.001, 0.004],
    "interaction":   [0.001, 0.001, 0.000, 0.002]
}, index=portfolio_returns.index)

# Link effects using Carino method
result = link(effects, portfolio_returns, benchmark_returns)

# View results
print(result.summary())
print(f"\nAllocation: {result['allocation']:.4%}")
print(f"Selection:  {result['selection']:.4%}")
print(f"k-factor: {result.k_factor:.4f}")
```

```python With Validation Options
from attriblink import link

# Disable effects sum validation for legacy data
result = link(
    effects, 
    portfolio_returns, 
    benchmark_returns,
    check_effects_sum=False  # Skip validation
)

# Or use strict mode to raise errors on mismatch
result = link(
    effects, 
    portfolio_returns, 
    benchmark_returns,
    strict=True  # Raises EffectsSumMismatchError if validation fails
)
```
</CodeGroup>

## Common Parameters

The `link()` function accepts these parameters:

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `effects` | `pd.DataFrame` | Required | Attribution effects with columns for each effect type |
| `portfolio_returns` | `pd.Series` | Required | Portfolio returns for each period |
| `benchmark_returns` | `pd.Series` | Required | Benchmark returns for each period |
| `method` | `str` | `"carino"` | Linking method (currently only "carino" supported) |
| `check_effects_sum` | `bool` | `True` | Validate effects sum to excess returns |
| `strict` | `bool` | `False` | Raise error on validation failure (vs. warning) |
